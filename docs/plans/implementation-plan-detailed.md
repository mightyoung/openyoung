## 五、质量门禁

### 5.1 验收标准

- [ ] 所有测试通过 `pytest tests/ -v`
- [ ] ruff检查通过 `ruff check src/`
- [ ] 类型检查通过 `mypy src/`
- [ ] 无新增警告

### 5.2 回滚计划

如果任何P0修复失败:
- 立即回滚到上一个稳定版本
- 分析失败原因
- 在下一轮迭代中重新修复

## 六、风险控制

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| YoungAgent拆分破坏现有功能 | 高 | 充分测试，覆盖率>80% |
| DI集成引入循环依赖 | 中 | 先分析依赖图，避免循环 |
| 并行修复冲突 | 低 | 不同Agent修改不同文件 |
