# Coding Standards Skill

## 描述
提供代码质量检查和编码规范能力。

## 核心原则

### 1. 代码质量
- 保持代码简洁清晰
- 遵循 DRY (Don't Repeat Yourself) 原则
- 适当的注释和文档

### 2. 命名规范
- 使用有意义的变量/函数名
- 遵循语言命名约定 (camelCase, PascalCase 等)
- 避免缩写，除非是通用缩写

### 3. 函数设计
- 单一职责 (SRP)
- 参数尽量少 (<= 3 个)
- 返回值明确

### 4. 错误处理
- 优雅处理异常
- 适当的错误信息
- 避免吞掉异常

## 检查清单

- [ ] 代码可读性
- [ ] 命名一致性
- [ ] 函数长度 (< 50 行)
- [ ] 参数数量
- [ ] 重复代码
- [ ] 错误处理
- [ ] 安全性
